## 2. HOOK useSiteData (src/hooks/useSiteData.ts)

O hook importa os dados do arquivo de configuração:

```typescript
import { useState } from 'react';
import { siteConfig } from '../../config/siteConfig';
import type { SiteData } from '../types/siteData';

export const useSiteData = () => {
  const [data] = useState<SiteData>(siteConfig);
  const [isLoading] = useState(false);
  
  return { data, isLoading, error: null };
};
```

### Componente SEO.tsx
Crie componente que injeta meta tags dinamicamente usando **document.title e manipulação nativa do DOM**.
**NÃO USE react-helmet-async** - causa conflitos com React 18.

```typescript
// Exemplo de SEO.tsx sem dependências problemáticas
import { useEffect } from 'react';

interface SEOProps {
  title: string;
  description?: string;
}

export const SEO = ({ title, description }: SEOProps) => {
  useEffect(() => {
    document.title = title;
    if (description) {
      const metaDesc = document.querySelector('meta[name="description"]');
      if (metaDesc) metaDesc.setAttribute('content', description);
    }
  }, [title, description]);
  
  return null;
};
```

📦 HOOK useSiteData (Simples e Direto)

O hook já está pronto para uso:

```typescript
// ═══════════════════════════════════════════════════════════════
// Hook: useSiteData
// Retorna dados do siteConfig.ts (arquivo local editável via Git)
// ═══════════════════════════════════════════════════════════════

import { useState } from 'react';
import { defaultSiteData } from '@/config/siteConfig';
import type { SiteData } from '@/types/siteData';

/**
 * Hook para acessar os dados do site.
 * 
 * Os dados vêm do arquivo config/siteConfig.ts que pode ser:
 * - Editado diretamente no repositório Git
 * - Editado via CMS conectado ao GitHub
 * 
 * Vantagens desta abordagem:
 * - Zero dependências de banco de dados
 * - Deploy automático via Git push
 * - Versionamento completo via Git
 * - Edições ficam no histórico do repositório
 */
export const useSiteData = () => {
  const [data] = useState<SiteData>(defaultSiteData);
  
  return { 
    data, 
    isLoading: false, 
    error: null 
  };
};

// Alias para compatibilidade
export const useSiteDataSimple = useSiteData;
```

⚠️ **SEM ESTE ARQUIVO O DEPLOY VAI FALHAR!**

Crie o arquivo `.npmrc` na RAIZ do projeto com este conteúdo:

```
legacy-peer-deps=true
```

**Por que isso é necessário?**
O React 19 não é compatível com algumas bibliotecas (react-helmet-async, etc).
Este arquivo resolve o erro: `ERESOLVE unable to resolve dependency tree`

**Como criar no GitHub:**
1. Acesse seu repositório no GitHub
2. Clique em "Add file" → "Create new file"
3. Nome do arquivo: `.npmrc` (com ponto no início!)
